# 🗄️ Database Design

The AI SEO Agents project uses PostgreSQL as the primary database.

The database is designed using a normalized relational model to efficiently store competitors, backlinks, AI analysis results, opportunity scores, workflow logs, and email logs.

---

# Database Tables

The project contains the following tables.

| Table | Purpose |
|--------|---------|
| competitors | Stores competitor companies monitored by the system. |
| backlink_sources | Stores unique backlink source domains. |
| competitor_backlinks | Stores backlinks collected for each competitor. |
| backlink_analysis | Stores AI-generated backlink analysis. |
| backlink_opportunities | Stores calculated backlink opportunity scores. |
| error_logs | Stores workflow execution errors. |
| email_logs | Stores email delivery history. |

---

# Table Details

## 1. competitors

**Purpose**

Stores the list of competitor companies monitored by the system.

**Primary Key**

- id

**Business Usage**

- Stores competitor name
- Domain
- Industry
- Status (ACTIVE / INACTIVE)
- Used by Agent 1 to fetch competitor backlinks

---

## 2. backlink_sources

**Purpose**

Stores unique backlink source domains collected from all competitors.

**Primary Key**

- id

**Business Usage**

- Prevents duplicate backlink domains
- Stores source domain
- Stores domain rating
- Shared across multiple competitors

---

## 3. competitor_backlinks

**Purpose**

Stores backlinks collected for each competitor.

**Primary Key**

- id

**Foreign Keys**

- competitor_id → competitors.id
- backlink_source_id → backlink_sources.id

**Business Usage**

- Stores backlink URL
- Stores anchor text
- Stores first_seen_date
- Stores last_seen_date
- Tracks whether the backlink has been analyzed by AI

---

## 4. backlink_analysis

**Purpose**

Stores AI-generated backlink analysis for each competitor backlink.

**Primary Key**

- id

**Foreign Keys**

- competitor_backlink_id → competitor_backlinks.id

**Business Usage**

- Stores relevance score
- Stores spam score
- Stores authority score
- Stores opportunity score
- Stores recommendation
- Stores analysis timestamp
- Generated by Agent 2

---

## 5. backlink_opportunities

**Purpose**

Stores the final backlink opportunity scores calculated for marketing teams.

**Primary Key**

- id

**Foreign Keys**

- analysis_id → backlink_analysis.id

**Business Usage**

- Stores domain rating score
- Stores relevance score
- Stores difficulty score
- Stores final score
- Stores priority (HIGH / MEDIUM / LOW)
- Stores suggested action
- Generated by Agent 3

---

## 6. error_logs

**Purpose**

Stores workflow execution errors for debugging and monitoring.

**Primary Key**

- id

**Business Usage**

- Stores workflow name
- Stores agent name
- Stores node name
- Stores error type
- Stores error message
- Stores input data
- Stores error timestamp

---

## 7. email_logs

**Purpose**

Stores email delivery history.

**Primary Key**

- id

**Business Usage**

- Stores recipient email
- Stores email subject
- Stores attachment name
- Stores delivery status
- Stores sent timestamp
- Stores failure reason (if any)

---
